Move a Solana validator identity
between hosts, securely.

End-to-end encrypted, single-shot transfer of validator-keypair.json (and the matching tower file) over an untrusted relay. The private key never leaves the operator's machines in the clear.

Designed around three properties

Untrusted relay

The relay only ever holds opaque ciphertext. The pairing code's hash prefix lets the sender locally verify the recipient's pubkey — a hostile relay cannot substitute keys.

Slashing-safe

Pre-flight reads the leader schedule and refuses to transfer inside an imminent leader window. Tower file moves with the keypair so the new host can set-identity --require-tower.

Memory hygiene

Keys live in sodium_malloc buffers — mlock'd, guard-paged, zeroized on every exit path including SIGINT. No copies on disk except the operator's two files.

Single-shot

Every session is one-time-use. Ciphertext is wiped from memory on first delivery, sessions TTL out in 5 minutes, the relay keeps no persistent state.

One static binary

C + libsodium + libcurl. Builds with make on Linux and macOS. No runtime, no installer, fits on a USB stick.

Self-hostable

The relay is a 200-line Fastify service. fly deploy, railway up, or docker run — pick one. No database, no shared backend.

The protocol in five lines

  • Receiver generates an ephemeral X25519 keypair and registers with the relay under a prefix = blake2b-512(pk)[0..8].
  • The 8-byte prefix is rendered as 6 BIP39 words and shown to the operator.
  • Sender types the words. The CLI fetches the recipient pubkey from the relay, re-derives the prefix, and refuses to upload on mismatch.
  • Sender seals (X25519 + XSalsa20-Poly1305) the validator keypair, tower file, and metadata to the recipient pubkey, and uploads.
  • Receiver long-polls, decrypts, verifies the embedded pubkey matches, writes 0600 files, optionally --require-tower's the swap.

API — the relay surface

GET /healthz

Liveness probe. Returns the count of live sessions (informational).

GET /v1/info

Runtime config: TTL, ciphertext cap, long-poll window. Used by vid-transfer doctor.

POST /v1/sessions

Register a recipient pubkey under prefix = blake2b-512(pk)[0..8]. The relay enforces the binding.

GET /v1/sessions/<prefix>

Look up a session. The CLI re-verifies blake2b(pk)[0..8] == prefix client-side.

PUT /v1/sessions/<prefix>/ciphertext

Upload sealed bundle. One-shot — second upload returns 409.

GET /v1/sessions/<prefix>/ciphertext?wait=1

Long-poll for delivery. Drained on first read; subsequent reads return 404.
